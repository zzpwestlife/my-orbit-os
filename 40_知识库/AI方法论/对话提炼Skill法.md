# 对话提炼 Skill 法

## 定义

将成功的多轮 AI 对话（尤其是经过反复调优才获得理想结果的过程）直接提炼为 Skill。无需专门设计，只需在对话结束后说一句话即可固化最佳实践。

## 核心方法

> "当你跟 Claude 聊天沟通把一个事情做完（比如对话很久调出很好的推特），然后说一句 '请把上面的推特写作方法写成 Skill' —— 马上你就有一套好的 skill 了。"

### 为什么有效

| 传统方式 | 对话提炼法 |
|---------|-----------|
| 先设计 Skill → 再测试 → 再迭代 | 先实战 → 成功后直接固化 |
| 想象需要什么功能 | 从真实成功案例中提取 |
| 可能不符合实际 workflow | 已经验证过的 workflow |

### 典型场景

- **内容创作** — 推特、博客、邮件的多轮打磨
- **代码调试** — 复杂的 bug 排查过程
- **数据分析** — 特定格式的报表生成
- **设计迭代** — UI/文案的多轮优化

## 操作步骤

### 1. 正常对话完成目标

```
用户: 帮我写个关于 AI 的推特
Claude: [初稿]
用户: 太正式了，要更口语化
Claude: [修改]
用户: 加点 emoji，更吸引人
Claude: [再改]
用户: 完美！就是这个感觉
```

### 2. 一句话固化

```
"请把上面的推特写作方法写成 Skill"
```

### 3. 下次直接使用

```
"/tweet 帮我写个关于远程工作的"
→ 自动应用之前调优的风格
```

## Skill 内容包含什么

被提炼的 Skill 会自动捕获：

- **初始策略** — 如何开始（如先问目标受众）
- **迭代模式** — 常见的修改方向（口语化→加 emoji→加 CTA）
- **质量标准** — 什么算"好"的最终结果
- **避坑指南** — 之前踩过的坑（如避免过度营销感）

## 对比：技术 Skill vs 对话 Skill

| 类型 | 来源 | 用途 |
|-----|------|------|
| **技术 Skill** | AI 执行脚本任务 | 自动化工具（替代 APP） |
| **对话 Skill** | 多轮对话优化内容 | 固化创作方法论 |

## 最佳实践

1. **趁热提炼** — 对话刚结束时立即固化，细节还记得
2. **命名清晰** — `/tweet`、`/blog`、`/email` 等直觉命名
3. **版本迭代** — 下次发现更好的方法时，更新 Skill
4. **组合使用** — 多个 Skill 形成个人内容生产线

## 示例 Skill 输出

```markdown
# Twitter Writing Skill

## 流程
1. 询问目标受众和核心信息
2. 生成 3 个不同角度的初稿
3. 应用风格：口语化 + 短句 + emoji
4. 检查：是否有 hook？是否有 CTA？
5. 输出最终版本 + 备选版本

## 风格指南
- 每句不超过 15 字
- 用 "你" 直接对话读者
- 数字具体化（"90%" 而非 "很多"）
- 结尾加问句或行动号召
```
