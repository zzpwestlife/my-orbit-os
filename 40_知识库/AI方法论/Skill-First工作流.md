# Skill-First 工作流

## 定义

用 AI Skill（脚本/自动化流程）替代传统 APP，通过"让 AI 先做一遍，再让它写成 Skill"的方法，快速构建个性化工具链。

## 核心洞察

> "Skill 越来越多地代替 APP 解决问题。写 skill 最简单的方法：让 AI 干一遍，然后让它把干的过程写成 skill。"

### 案例：干掉 Topaz 系列

| 传统方案 | Skill 方案 |
|---------|-----------|
| Topaz Gigapixel AI（图片放大） | 自动判断类型 → 调用对应处理流程 |
| Topaz Video Enhance AI（视频增强） | 统一入口，智能路由 |
| 手动操作 GUI | 一句话指令，后台自动执行 |

**结果**：一个 Skill 替代两个付费 APP，零手动操作。

## Skill 创建方法论

### 三步法

```
1️⃣ 让 AI 执行一次完整任务
   ↓
2️⃣ 让它把过程写成可复用的 Skill
   ↓
3️⃣ 下次直接调用 Skill
```

### Prompt 模板

**第一步：执行任务**
```
请帮我完成 [具体任务]，记录你使用的：
- 工具/命令
- 参数配置
- 判断逻辑
- 错误处理方式
```

**第二步：生成 Skill**
```
基于刚才的过程，请创建一个 Skill：
- 输入：用户提供的文件/参数
- 处理：自动判断类型并执行对应流程
- 输出：处理后的结果
- 保存为可复用脚本
```

## Skill 优势矩阵

| 维度 | 传统 APP | AI Skill |
|-----|---------|---------|
| 成本 | 订阅/买断制 | 一次性开发，永久使用 |
| 灵活性 | 受限于 UI | 完全可定制 |
| 集成度 | 独立运行 | 融入工作流 |
| 学习成本 | 熟悉新界面 | 自然语言交互 |
| 自动化 | 有限/无 | 完全可自动化 |

## 适合 Skill 替代的场景

✅ **高潜力场景**：
- 格式转换（视频/音频/图片/文档）
- 批量重命名/整理文件
- 定时任务（备份、清理、同步）
- 数据抓取与处理
- 图片/视频预处理

⚠️ **暂不替代**：
- 专业创意软件（Photoshop、Final Cut）
- 高度依赖硬件加速的渲染
- 合规审计严格的场景

## Skill 设计原则

1. **智能路由** — 自动判断输入类型，选择对应处理流程
2. **统一入口** — 一个命令处理多种场景
3. **错误兜底** — 失败时提供清晰的反馈和备选方案
4. **渐进增强** — 从简单脚本开始，逐步添加功能

## 进阶：Skill 组合

```
Skill A: 下载视频 → Skill B: 画质增强 → Skill C: 转码压缩
         ↓
      组合成 Pipeline：「一键视频处理」
```
